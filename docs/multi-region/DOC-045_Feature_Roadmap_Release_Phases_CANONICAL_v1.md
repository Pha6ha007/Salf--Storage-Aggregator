# 1. Цели MVP

## 1.1. Основные продуктовые цели

**Note:** *Detailed functional requirements defined in DOC-001.*

MVP (Minimum Viable Product) Self-Storage Aggregator создаётся для решения ключевых проблем рынка складских помещений в России и предоставления пользователям и операторам складов эффективного инструмента взаимодействия.

### Главные цели продукта:

**1. Упростить поиск складских помещений**
- Агрегировать предложения от разных операторов на единой платформе
- Обеспечить быстрый поиск по ключевым параметрам (локация, размер, цена)
- Предоставить прозрачную информацию о доступности боксов в режиме реального времени

**2. Обеспечить прозрачное сравнение условий**
- Стандартизировать представление информации о складах и боксах
- Предоставить инструменты для сравнения до 4 складов одновременно
- Показать реальные цены с учётом скидок и дополнительных услуг

**3. Создать удобный процесс бронирования**
- Упростить процесс до 3 шагов: выбор → заполнение формы → подтверждение
- Обеспечить мгновенное получение подтверждения бронирования
- Предоставить понятный личный кабинет для отслеживания заявок

**4. Повысить загрузку складов для операторов**
- Предоставить операторам доступ к широкой аудитории арендаторов
- Обеспечить простые инструменты управления складами и боксами
- Снизить барьер входа на рынок для новых операторов

**5. Обеспечить качество сервиса**
- Внедрить систему модерации складов и операторов
- Создать механизмы контроля качества информации
- Обеспечить быструю обработку заявок операторами

### Ключевые продуктовые метрики MVP:

**Note:** *These are aspirational targets, not contractual obligations.*

| Метрика | Целевое значение | Период измерения |
|---------|------------------|------------------|
| Время от поиска до бронирования | < 5 минут | На одну сессию |
| Конверсия поиск → просмотр склада | > 15% | Месяц |
| Конверсия просмотр → бронирование | > 3% | Месяц |
| Время обработки заявки оператором | < 24 часа | На заявку |
| Качество информации о складах | > 90% актуальности | Аудит |

## 1.2. Цели по ролям пользователей

MVP обслуживает три основные роли: Арендатор (User), Оператор склада (Operator) и Администратор (Admin). Каждая роль имеет специфические цели и ожидания.

### 1.2.1. User (Арендатор)

**Профиль пользователя:**
- Физические лица, нуждающиеся в хранении личных вещей (переезд, ремонт, сезонные вещи)
- Малый бизнес, требующий складских площадей для товаров
- Владельцы транспорта, ищущие паркинг/бокс для авто/мото

**Цели арендатора в MVP:**

1. **Доступность информации**
   - Видеть все доступные склады в нужном районе
   - Получать актуальную информацию о наличии свободных боксов
   - Понимать полную стоимость аренды (включая дополнительные услуги)

2. **Понятность процесса**
   - Интуитивный интерфейс без необходимости обучения
   - Понятная терминология (избегание профессионального жаргона)
   - Четкие инструкции на каждом шаге бронирования

3. **Скорость принятия решения**
   - Быстрая фильтрация по ключевым параметрам
   - Наглядное сравнение вариантов
   - AI-рекомендации для подбора подходящего размера бокса

4. **Уверенность в выборе**
   - Полная информация о складе (фото, описание, условия)
   - Информация об операторе и его надёжности
   - Понятные условия аренды и отмены

5. **Удобство коммуникации**
   - Быстрое получение ответов на вопросы через чат/форму
   - Уведомления о статусе заявки
   - Возможность связаться с оператором напрямую

**Ожидаемый пользовательский опыт:**
```
Поиск (30 сек) → Сравнение 2-3 вариантов (2 мин) → 
Детальный просмотр (1 мин) → Бронирование (1.5 мин) → 
Подтверждение от оператора (до 24 часов)
```

**Success Criteria для User:**
- Нашёл подходящий склад за < 5 минут
- Понял стоимость и условия без звонка оператору
- Получил подтверждение бронирования в течение суток
- Может отследить статус заявки в личном кабинете

---

### 1.2.2. Operator (Оператор склада)

**Профиль оператора:**
- Малые и средние компании, управляющие 1-5 складами
- Крупные операторы с сетью складов (будущая целевая аудитория)
- Новые игроки рынка, запускающие первые объекты

**Цели оператора в MVP:**

1. **Привлечение клиентов**
   - Получить доступ к широкой базе потенциальных арендаторов
   - Повысить видимость своих складов через агрегатор
   - Конкурировать с крупными игроками на равных

2. **Простота управления**
   - Быстро добавить склад и боксы в систему
   - Легко обновлять информацию о доступности и ценах
   - Управлять заявками из единого интерфейса

3. **Эффективная обработка заявок**
   - Получать уведомления о новых бронированиях
   - Быстро подтверждать/отклонять заявки
   - Коммуницировать с клиентами через платформу

4. **Контроль качества**
   - Получать обратную связь от клиентов
   - Мониторить статистику просмотров и бронирований
   - Понимать конкурентное окружение

5. **Минимизация барьеров входа**
   - Простая регистрация и онбординг
   - Понятный интерфейс без необходимости технических знаний
   - Бесплатное размещение базового объявления

**Ожидаемый операторский опыт:**
```
Регистрация (5 мин) → Верификация админом (1-2 дня) → 
Добавление склада (10 мин) → Добавление боксов (15 мин) → 
Получение первой заявки (в течение недели) → 
Обработка заявки (в течение дня)
```

**Success Criteria для Operator:**
- Добавил склад и боксы за < 30 минут
- Получил первую заявку в течение недели после активации
- Обрабатывает заявки в течение 24 часов
- Понимает статистику по своим объектам

---

### 1.2.3. Admin (Администратор платформы)

**Профиль администратора:**
- Внутренняя команда платформы
- Ответственные за качество контента и модерацию
- Технические специалисты, мониторящие работу системы

**Цели администратора в MVP:**

1. **Контроль качества**
   - Модерировать новых операторов перед допуском к платформе
   - Проверять информацию о складах на соответствие стандартам
   - Отслеживать качество фотографий и описаний

2. **Безопасность платформы**
   - Предотвращать появление мошеннических объявлений
   - Блокировать недобросовестных операторов
   - Мониторить подозрительную активность

3. **Поддержка пользователей**
   - Разрешать конфликтные ситуации между арендаторами и операторами
   - Помогать операторам в работе с платформой
   - Отвечать на вопросы пользователей

4. **Аналитика и улучшение**
   - Отслеживать ключевые метрики платформы
   - Выявлять проблемные места в пользовательском опыте
   - Предлагать улучшения на основе данных

5. **Управление системой**
   - Настраивать параметры платформы
   - Управлять контентом (FAQ, правила, условия)
   - Мониторить техническое состояние системы

**Ожидаемый административный процесс:**
```
Новая регистрация оператора → Проверка документов (1-2 часа) → 
Верификация (одобрить/отклонить) → 
Мониторинг нового склада (первые 2 недели) → 
Регулярные аудиты (раз в месяц)
```

**Success Criteria для Admin:**
- Проверяет новых операторов за < 2 рабочих дней
- Модерирует склады за < 24 часа
- Разрешает конфликты за < 3 рабочих дня
- Поддерживает качество контента на уровне > 90%

## 3.1. AI Box Size Recommendation (Подбор размера бокса)

**Приоритет:** HIGH  
**Технология:** Claude API + Rule-based система

### Описание функциональности

AI-помощник помогает пользователям определить оптимальный размер бокса на основе типа и количества вещей.

### User Flow

1. **Триггер:** Кнопка "Подобрать размер с помощью AI" на странице склада
2. **Интерфейс опроса:** 3 вопроса о типе вещей, объёме, особых требованиях
3. **AI анализ:** Claude API рекомендует 2-3 подходящих варианта
4. **Результат:** Показ рекомендованных боксов с пояснениями

### Технические детали

**API Integration:**
```
POST /api/v1/ai/recommend-box
Request: warehouse_id, items, volume, requirements
Response: recommendations (box_id, confidence, reason, price)
```

**Fallback:** При недоступности Claude API → rule-based система

---

## 3.2. AI Price Analytics (Рекомендации по цене)

**Приоритет:** MEDIUM  
**Технология:** Pandas + Scikit-learn

### Описание функциональности

AI анализирует рыночные цены и даёт операторам рекомендации по оптимальному ценообразованию.

### Как работает

1. **Сбор данных:** Цены всех боксов в городе, особенности складов, заполненность
2. **Анализ:** Расчёт медианной цены, корректировка на особенности
3. **Рекомендация:** Предложение оптимальной цены с прогнозом эффекта

**Algorithm:**
1. Собрать боксы схожего размера в радиусе 5 км
2. Рассчитать median и percentiles
3. Применить коэффициенты за особенности
4. Если цена > P75 → рекомендовать снижение

---

## 3.3. AI Chat Assistant (Базовый чат-помощник)

**Приоритет:** MEDIUM  
**Технология:** Claude API + RAG

### Описание функциональности

Чат-помощник для пользователей и операторов, отвечающий на частые вопросы.

### Scope для MVP (ограниченная версия)

**Что умеет:**
1. Отвечать на FAQ
2. Помогать с навигацией
3. Поиск складов

**Что НЕ умеет в MVP:**
- Создавать заявки (только направляет на форму)
- Изменять данные
- Обрабатывать платежи

### Технические детали

**RAG Pipeline:**
1. Knowledge Base: FAQ, User manual, Terms
2. Vector Search: Embedding модель, Vector DB
3. Claude Generation: Prompt с контекстом, краткие ответы

**Rate Limiting:** 10 сообщений в минуту на пользователя

---

## 3.4. AI Content Suggestions (Улучшение описаний)

**Приоритет:** LOW  
**Технология:** Claude API (text generation)

### Описание функциональности

AI помогает операторам создавать качественные описания складов.

### Как работает

1. **Триггер:** Оператор добавляет склад, видит кнопку "Сгенерировать описание"
2. **AI генерация:** Claude анализирует особенности склада, генерирует описание
3. **Редактирование:** Оператор может отредактировать текст
4. **Применение:** Сохранение описания

**Claude Prompt:** Генерация профессионального описания на русском языке с учётом особенностей, локации, SEO-keywords
